# Roadmap

## Fase 0: Documentacion

Estado actual.

Objetivo:

- Definir producto.
- Definir alcance MVP.
- Definir arquitectura tecnica.
- Definir modelo de datos.
- Definir API.
- Definir seguridad.
- Definir pruebas y despliegue.

## Fase 1: MVP beta privada

Objetivo:

- Construir app usable para primeros clientes beta.

Incluye:

- Login privado.
- Un usuario por negocio.
- Clientes.
- Facturas internas.
- Pagos.
- Saldos.
- Dashboard.
- Comprobante imprimible.
- Swagger.
- RLS.
- Deploy en Vercel + Supabase.

## Fase 2: Validacion comercial

Objetivo:

- Probar con negocios reales.
- Medir si el producto resuelve el problema.

Actividades:

- Dar acceso a estetica/boutique.
- Observar uso durante varios dias.
- Registrar feedback.
- Identificar fricciones.
- Medir necesidad de PDF, reportes, WhatsApp o recordatorios.
- Definir precio tentativo.

## Fase 3: Producto vendible

Objetivo:

- Preparar el producto para cobrar.

Posibles funcionalidades:

- Registro publico.
- Onboarding.
- Planes.
- Limites por plan.
- Suscripcion.
- PDF generado.
- Correos transaccionales.
- Mejoras visuales.
- Plantilla de comprobante.

## Fase 4: Colaboracion

Objetivo:

- Soportar negocios con mas de una persona.

Funcionalidades:

- Multiples usuarios por negocio.
- Invitaciones.
- Roles.
- Permisos.
- Auditoria basica.

## Fase 5: Automatizacion

Objetivo:

- Reducir trabajo manual del negocio.

Funcionalidades:

- Recordatorios de pago.
- Facturas recurrentes.
- Envio por correo.
- Envio por WhatsApp.
- Exportacion a Excel.
- Reportes periodicos.

## Fase 6: Operacion avanzada

Objetivo:

- Ampliar casos de uso segun validacion.

Posibles modulos:

- Catalogo de productos/servicios.
- Inventario basico.
- Reportes avanzados.
- Integraciones contables.
- Facturacion electronica en una etapa futura si el mercado lo exige.

## Criterio para avanzar de fase

No avanzar por cantidad de funcionalidades, sino por evidencia:

- Clientes usan la app sin acompanamiento constante.
- El dashboard les ayuda a tomar decisiones.
- Registran pagos y facturas de forma recurrente.
- Piden mejoras especificas.
- Estan dispuestos a pagar.
